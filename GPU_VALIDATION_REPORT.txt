PART 6: ROOT CAUSE ANALYSIS
================================================================================

Why is GPU not helping?

1. OPERATION SIZE vs OVERHEAD:
   - Color analysis operations (mean/weighted mean) are small
   - GPU transfer overhead (CPU→GPU→CPU) likely exceeds computation time
   - Estimated transfer cost: 5-10ms per operation
   - Actual computation time: <1ms on either CPU or GPU

2. BOTTLENECK NOT IN COLOR ANALYSIS:
   - Primary cost breakdown (estimated):
     * Screen capture (DXGI): 40-50ms of total latency
     * Color analysis (GPU or CPU): 1-2ms
     * Network I/O to LED controller: 5-10ms
   - Color analysis is NOT the bottleneck

3. GPU INITIALIZATION COST:
   - CuPy/CUDA initialization has overhead
   - First operations slower due to kernel compilation
   - Profile shows GPU + CPU fallback path may be executing

4. CUPY LIMITATIONS:
   - CuPy resize() delegates to OpenCV CPU anyway
   - No benefit for small array operations
   - CuPy better suited for large matrix operations
